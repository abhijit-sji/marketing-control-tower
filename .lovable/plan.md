# Live-DB Bundle Export

Pull everything from the **live database** (not source files) and save as ordered SQL/CSV bundles under `/mnt/documents/db-bundle/` for you to download and apply to the new Supabase project.

## What gets exported

Each file is generated by querying Postgres system catalogs via `psql`.

| Order | File | Contents | Source |
|------:|------|----------|--------|
| 00 | `00_extensions.sql` | `CREATE EXTENSION` for every non-default extension | `pg_extension` |
| 01 | `01_tables.sql` | `CREATE TABLE` DDL for all `public` tables (columns, types, defaults, NOT NULL) | `pg_class` + `pg_attribute` |
| 02 | `02_constraints.sql` | PKs, FKs, UNIQUE, CHECK constraints | `pg_constraint` via `pg_get_constraintdef` |
| 03 | `03_indexes.sql` | All non-PK indexes on `public` tables | `pg_indexes` |
| 04 | `04_functions.sql` | All `public` functions (incl. `has_role`, `user_has_brand_access`, etc.) | `pg_proc` via `pg_get_functiondef` |
| 05 | `05_triggers.sql` | All triggers on `public` tables | `pg_trigger` via `pg_get_triggerdef` |
| 06 | `06_grants.sql` | Role grants on every `public` table | `information_schema.role_table_grants` |
| 07 | `07_rls_enable.sql` | `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` per table | `pg_class.relrowsecurity` |
| 08 | `08_policies.sql` | All RLS policies (roles, cmd, USING, WITH CHECK) | `pg_policies` |
| 09 | `09_sequences.sql` | Sequence definitions + ownership | `pg_sequences` |
| 10 | `10_views.sql` | Views in `public` | `pg_views` |
| 11 | `11_data/*.csv` | One CSV per non-empty public table (data rows) | `COPY ... TO STDOUT CSV HEADER` |
| 12 | `12_data_load.sh` | Helper script: `psql \copy` each CSV back into the new DB in FK-safe order | generated |
| 13 | `13_edge_functions.txt` | List of edge function names + sizes pulled from `supabase/functions/` (DB does not store function code; this lists what to deploy) | filesystem listing of repo |
| 14 | `14_apply_all.sh` | Runs files 00→10 against a target DB, then `12_data_load.sh` | generated |

## Order to apply on the new project

```
00_extensions.sql
01_tables.sql
02_constraints.sql
03_indexes.sql
09_sequences.sql
04_functions.sql
05_triggers.sql
10_views.sql
06_grants.sql
07_rls_enable.sql
08_policies.sql
12_data_load.sh        # loads CSVs from 11_data/
```
Then deploy edge functions listed in `13_edge_functions.txt`.

## Notes & caveats

- Bundles are pulled from the **live DB schema** as it exists right now — they reflect every migration that's already been applied, including hotfixes you did directly.
- `auth.*`, `storage.*`, `realtime.*`, `supabase_functions.*`, `vault.*` schemas are **excluded** (managed by Supabase).
- Auth users are **not** exported (cannot read `auth.users` passwords). Recreate demo/admin users on the new project via dashboard or the `seed-demo-users` edge function after deploy.
- Storage buckets: project currently has **0** — nothing to export.
- Edge function source code is not stored in the DB; `13_edge_functions.txt` will tell you which functions to `supabase functions deploy` from the repo.
- Tables with very large rows (embeddings, knowledge files) will produce large CSVs — `11_data/` may be hundreds of MB. I'll print a size report at the end.
- Sequence current values are captured via `setval` lines appended to `09_sequences.sql` so IDs continue from the right place.

## How to test

1. After I generate the bundles, you'll get a directory listing with file sizes.
2. On the new project, run `bash 14_apply_all.sh` pointing at the new DB URL.
3. Verify table count, row counts on key tables (`brands`, `ai_agents`, `user_roles`), and that RLS is enabled on every table.
4. Login with a recreated demo user and load `/adminpanel` — confirms grants + policies work.

Approve and switch to build mode so I can run the export.
